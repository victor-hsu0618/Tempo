# Tempo 轉換設計改進

## 改進內容

### 1. **完整的"從...轉到..."設計**

**之前:** 轉換只能自動進行到下一個時間簽名
```
當前: 4/4
點擊「添加轉換」→ 自動轉到 2/4（無法選擇）
```

**現在:** 用戶可以明確選擇源和目標時間簽名
```
從: [選擇下拉菜單]
  ├─ 2/4
  ├─ 3/4  
  ├─ 4/4 ← 可任意選擇
  └─ 6/8

轉到: [選擇下拉菜單]
  ├─ 2/4
  ├─ 3/4 ← 可任意選擇
  ├─ 4/4
  └─ 6/8

第幾個小節轉換: [輸入框]
  └─ 例：2（表示第 2 小節開始轉換）
```

## 核心功能

### 1. 明確的轉換源和目標
- **從**: 當前練習的時間簽名
- **轉到**: 要轉換到的時間簽名
- **轉換點**: 第幾個小節開始轉換

### 2. 自動驗證
```javascript
// 添加轉換時自動驗證
if (transition.from !== currentTimeSignature) {
  // 自動切換到轉換的源時間簽名
  setTimeSignature(transition.from);
}
```

### 3. 多轉換支持
```
同一會話支持多個連續轉換：
2/4 → 3/4 (第 2 小節)
3/4 → 4/4 (第 5 小節)
4/4 → 6/8 (第 9 小節)
```

**3/4 → 4/4 轉換訓練**

```
會話 1: BPM 100

會話 2: BPM 120

會話 3: BPM 140
  ├─ 第 2 小節: 2/4 → 3/4
```

## 技術實現

### 類型定義
```typescript
export interface TimeSignatureTransition {
  from: TimeSignature;      // ← 明確的源拍號
  to: TimeSignature;        // ← 明確的目標拍號
  measureAtTransition: number; // ← 第幾小節轉換
  warningMeasure?: number;  // ← 提前警告的小節數
}
```

### UI 狀態管理
```javascript
const [transitionFromIdx, setTransitionFromIdx] = useState(0);  // 源索引
const [transitionToIdx, setTransitionToIdx] = useState(1);      // 目標索引
const [transitionBeat, setTransitionBeat] = useState('2');      // 轉換小節
```

### 表單樣式改進
- ✓ 清晰的分組標籤
- ✓ 下拉菜單選擇器
- ✓ 按鈕預覽轉換
- ✓ 實時驗證

## 使用流程

### 步驟 1: 選擇練習時間簽名
```
點擊時間簽名按鈕: 4/4
```

### 步驟 2: 添加轉換
```
點擊 "+ 添加轉換"
├─ 從: 選擇 4/4
├─ 轉到: 選擇 2/3
└─ 第幾小節: 輸入 2
```

### 步驟 3: 確認轉換
```
點擊「確認 4/4 → 2/3」
```

### 步驟 4: 開始練習
```
點擊播放按鈕開始練習
- 播放 2 個 4/4 小節
- 第 3 個小節開始轉換到 2/3
- 用戶需要在轉換點準確擊鼓
```
